# Task Tracker API

Мини-проект, разработанный в рамках лабораторной работы 9–10.

## Описание проекта

**Task Tracker API** — это веб-приложение для управления задачами.  
Сервис позволяет создавать, просматривать, изменять и удалять задачи, а также отмечать их как выполненные.

Проект реализован в формате MVP и предназначен для демонстрации базовой серверной логики, работы с PostgreSQL и тестирования API.

## Функциональные возможности

- создание задачи
- просмотр всех задач
- просмотр задачи по ID
- редактирование задачи
- удаление задачи
- изменение статуса выполнения задачи

## Используемая база данных

В проекте используется **PostgreSQL**.  
Данные о задачах хранятся в таблице `tasks`.

Основные поля таблицы:
- `id` — идентификатор задачи
- `title` — название задачи
- `description` — описание задачи
- `is_done` — статус выполнения
- `created_at` — дата и время создания

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/alinaaved/task_tracker
cd task-tracker
```

### 2. Создать и активировать виртуальное окружение

#### macOS / Linux

```bash
python3 -m venv venv
source venv/bin/activate
```

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Создать базу данных PostgreSQL

Открыть `psql` и выполнить:

```sql
CREATE DATABASE task_tracker;
```

### 5. Настроить файл `.env`

Пример содержимого:

```env
DATABASE_URL=postgresql:///task_tracker
```

При необходимости строка подключения может быть изменена в соответствии с локальной конфигурацией PostgreSQL.

### 6. Запустить приложение

```bash
uvicorn app.main:app --reload
```

## Примеры запросов

### Создание задачи

`POST /tasks`

```json
{
  "title": "Сделать лабораторную",
  "description": "Подготовить MVP проекта"
}
```

### Обновление задачи

`PUT /tasks/{task_id}`

```json
{
  "title": "Сделать лабораторную работу",
  "description": "Подготовить MVP и README",
  "is_done": true
}
```
